# 编码要求

- 使用 TypeScript 作为开发语言，使用 ReactJS 和 Tailwind CSS 作为框架，使用 Zustand 做数据存储，使用 Ant Design 作为通用组件库。
- 在 Ant Design 中有功能类似的子组件，可以用 Ant Design 实现，组件的使用有疑问可以使用 context7 MCP 读取相关文档。
- 页面复杂时可以做功能子组件的拆分，不要过度拆分。
- Tailwind CSS 样式不满足时，可以使用样式文件完成 UI。
- 多组件共用数据可以使用 Zustand 存储，不要过度封装。
- 可复用逻辑可以使用 hooks 抽离，不要过度抽离。
- 保持代码风格简练，不要做复杂的封装。
- 先梳理需求功能，功能逻辑走不通时可以向我提问，没有问题后再开始编写代码，严禁编写不存在的功能逻辑。
- 有关接口的使用可以先自定义，放在 src/apis 文件夹下，然后 mock 数据，后续根据真实接口调整。
- 每个功能模块不是完全独立的，需要根据项目架构梳理模块之间的逻辑关系。
