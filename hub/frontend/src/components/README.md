# README 编写规范（经验总结）

基于本次实现经验，总结组件 README.md 应包含的核心内容，方便后续编写其他组件的文档：

## 1. 组件介绍

- **组件用途**：说明组件的主要功能和用途
- **整体结构**：如果组件包含多个子部分，说明各部分的关系和职责
- **使用场景**：简要说明在什么场景下使用该组件

## 2. 组件结构

- **文件列表**：列出组件目录下的所有文件及其职责
- **文件说明**：每个文件的作用和包含的内容类型（类型定义、主组件、子组件、工具函数等）

## 3. 交互设计

### 3.1 Figma 设计稿链接

- **必须提供**：每个主要 UI 部分的 Figma 链接
- **格式规范**：使用 Markdown 链接格式，标注清楚每个链接对应的部分
- **示例**：
  ```markdown
  - **基本信息**: [Figma 链接](url)
  - **业务知识网络**: [Figma 链接](url)
  ```

### 3.2 布局结构

- **整体布局**：描述组件的整体布局方式（上下、左右、网格等）
- **关键尺寸**：重要的宽度、高度、间距等（如抽屉宽度、菜单宽度）
- **响应式说明**：如果有响应式设计，说明不同屏幕下的表现

### 3.3 交互行为

- **用户操作**：点击、切换、展开/收起等交互行为
- **状态变化**：不同状态下的 UI 表现（选中、hover、loading 等）
- **默认行为**：组件初始化时的默认状态和行为

### 3.4 外部链接/跳转

- **外部链接**：如果组件包含跳转到其他平台的链接，说明链接格式和参数
- **链接文案**：说明链接的显示文案

## 4. 代码实现

### 4.1 Props 接口

- **接口定义**：提供完整的 TypeScript 接口定义
- **参数说明**：每个参数的类型、是否必填、用途说明
- **示例**：
  ```typescript
  interface ComponentProps {
    /** 参数说明 */
    prop1: string
    /** 参数说明 */
    prop2?: number
  }
  ```

### 4.2 默认行为

- **初始化状态**：组件首次渲染时的默认状态
- **默认值**：如果有默认值，说明默认值是什么

### 4.3 数据加载

- **数据来源**：数据来自哪个接口
- **接口位置**：接口定义在哪个文件中（如 `/src/apis/xxx.ts`）
- **加载时机**：什么时候触发数据加载（组件挂载、用户操作、状态变化等）
- **加载状态**：如何处理 loading 和 error 状态

### 4.4 样式规范

- **样式方案**：主要使用什么样式方案（Tailwind、CSS Modules、styled-components 等）
- **关键样式**：重要的颜色、尺寸、间距等设计 token
- **特殊样式**：如果有特殊的样式需求（如渐变、阴影、动画等），需要说明

### 4.5 性能优化

- **优化措施**：使用了哪些性能优化手段（memo、useMemo、useCallback 等）
- **注意事项**：需要注意的性能问题

## 5. 注意事项

- **特殊逻辑**：组件中的特殊处理逻辑
- **边界情况**：空数据、错误状态等边界情况的处理
- **依赖关系**：组件依赖的其他组件或工具
- **已知问题**：如果有已知问题或限制，需要说明

## 6. 可选内容

根据组件复杂度，可以添加：

- **使用示例**：简单的代码使用示例
- **API 文档**：如果组件暴露了方法或事件，说明 API
- **测试说明**：如果有特殊测试需求，说明测试要点
- **更新日志**：如果组件有重要变更，记录变更历史

---

## 编写建议

1. **结构清晰**：使用 Markdown 标题层级组织内容，便于快速定位
2. **信息完整**：确保包含足够的信息，让开发者能够独立实现组件
3. **格式统一**：保持代码块、链接、列表等格式的一致性
4. **及时更新**：组件实现变更时，及时更新 README
5. **Figma 链接必须**：UI 组件必须提供 Figma 设计稿链接，这是实现准确 UI 的关键
