# 上传应用 - 填写信息页面设计文档

## 1. 模块定义

本页面为应用上传流程的第二步"填写信息"页面，主要包含以下模块：

1. **顶部导航栏模块**：包含 Logo、面包屑导航、用户信息
2. **步骤条模块**：显示当前流程进度（3 步流程）
3. **表单内容模块**：应用信息填写表单（白色卡片容器）
4. **操作按钮模块**：表单底部操作按钮

## 4. 交互设计

### 4.1 表单交互

**应用名称输入框**

- **聚焦状态**：边框颜色变为 `#1677FF`（蓝色）
- **输入验证**：必填项，失去焦点时验证是否为空
- **错误状态**：显示错误提示，边框颜色变为 `#FF4D4F`（红色）

**应用描述文本域**

- **聚焦状态**：边框颜色变为 `#1677FF`（蓝色）
- **字数限制**：最大 200 字，实时显示字数统计
- **超出限制**：字数统计文字变为红色，禁止继续输入

**应用图标上传**

- **点击上传按钮**：触发文件选择对话框
- **文件限制**：
  - 格式：JPG、PNG
  - 推荐尺寸：114px × 114px
- **上传后显示**：
  - 上传框内显示预览图
  - 图片自动缩放适配 104px × 104px 容器
  - 保持宽高比，居中显示
- **悬停状态**：上传按钮背景色变为 `rgba(0, 0, 0, 0.05)`
- **点击状态**：上传按钮背景色变为 `rgba(0, 0, 0, 0.1)`

### 4.2 按钮交互

**下一步按钮**

- **默认状态**：背景色 `rgba(18, 110, 227, 0.25)`
- **悬停状态**：背景色变为 `rgba(18, 110, 227, 0.35)`
- **点击状态**：背景色变为 `rgba(18, 110, 227, 0.45)`
- **禁用状态**：
  - 当应用名称为空时禁用
  - 背景色变为 `rgba(0, 0, 0, 0.06)`
  - 文字颜色变为 `rgba(0, 0, 0, 0.25)`
- **点击行为**：验证表单，通过后跳转到下一步"上传应用安装包"

**上传图片按钮**

- **悬停状态**：背景色变为 `rgba(0, 0, 0, 0.05)`
- **点击状态**：背景色变为 `rgba(0, 0, 0, 0.1)`
- **点击行为**：触发文件选择对话框

### 4.3 步骤条交互

- **当前步骤**：步骤 1 为激活状态，显示蓝色背景和白色文字
- **未完成步骤**：步骤 2、3 为未激活状态，显示灰色背景和灰色文字
- **步骤不可点击**：当前流程为线性流程，已完成步骤不可回退

### 4.4 顶部导航交互

**面包屑导航**

- **首页图标**：点击返回首页
- **悬停状态**：背景色变为 `rgba(0, 0, 0, 0.08)`

**用户信息下拉**

- **点击用户区域**：展开下拉菜单（设计图中未显示菜单内容）
- **悬停状态**：背景色变为 `rgba(0, 0, 0, 0.05)`

## 6. 响应式设计建议

- **最小宽度**：1366px（当前设计宽度）
- **内容区域**：固定宽度 1020px，居中显示
- **移动端适配**：需要重新设计布局，考虑垂直堆叠和全宽输入框

---

## 7. 可访问性建议

- **表单标签**：使用 `<label>` 标签关联输入框
- **必填标识**：使用 `aria-required="true"` 属性
- **错误提示**：使用 `aria-invalid` 和 `aria-describedby` 属性
- **按钮状态**：使用 `aria-disabled` 属性标识禁用状态
- **步骤条**：使用 `aria-current="step"` 标识当前步骤
